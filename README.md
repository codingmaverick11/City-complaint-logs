# Municipal Complaint Analytics Dashboard

## Project Overview

This project analyzes over 16,000 municipal complaint records to identify civic issue hotspots, complaint patterns, and agency workload distribution. The objective was to transform raw complaint data into actionable insights using Python, MySQL, SQL, and Power BI.

---

## Business Problem

City administrations receive thousands of citizen complaints related to infrastructure, sanitation, roads, water supply, and other civic services. Understanding where complaints are concentrated and which agencies handle the highest workload can help improve resource allocation and service delivery.

---

## Dataset

* Source: Public municipal complaint dataset
* Records: 16,000+ complaints
* Fields: Complaint category, sub-category, ward, civic agency, status, creation date, and location information

---

## Tools & Technologies

* Python (Pandas)
* MySQL
* SQL
* Power BI
* Power Query

---

## Project Workflow

1. Imported raw complaint data.
2. Cleaned and standardized data using Python (Pandas).
3. Handled missing values and corrected data types.
4. Loaded cleaned data into MySQL.
5. Performed SQL analysis to identify trends and patterns.
6. Built interactive Power BI dashboards for visualization and decision support.

---

## Key Analyses Performed

* Complaint volume by category
* Complaint hotspots by ward
* Civic agency workload analysis
* Complaint status distribution
* Monthly complaint trend analysis

---

## Dashboard Features

* Interactive filtering by category, ward, and complaint status
* Top complaint categories visualization
* Ward-wise complaint distribution
* Civic agency workload monitoring
* Complaint resolution tracking

---

## Key Insights

* Complaint volumes were concentrated in a limited number of categories.
* Certain wards consistently generated higher complaint volumes than others.
* Workload distribution across civic agencies was uneven.
* Complaint resolution status varied significantly across issue categories.
